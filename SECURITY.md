# Security Policy

## Supported Versions

DFIR-Lab is maintained for the current codebase in this repository. There is no formal support lifecycle documented for older releases in this repository.

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly by opening a GitHub issue titled `SECURITY_REPORT`.

Provide the following information when possible:

- A clear description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The affected component or endpoint.
- Any mitigation or remediation suggestions.

## Responsible Disclosure

Do not publish unpatched vulnerabilities publicly before allowing time for a fix.

## Security Contact

If you need to contact the maintainers privately, use the repository's issue tracker and flag the issue as a security report.
